最方便的让图片能显示的方法

我自己的弱智wordpress把媒体库的文件url搞得乱七八糟啊，而且还不能改（数据库记录已经全部改掉了，但是url还是错的，那是真的牛*）。

那么最简单的方法就是把wordpress这个网站的domain写一条重定向，把访问wordpress.huaj.tk的wp-content/uploads的所有请求都rewrite到static.huaj.tk上，同时把写死了的媒体库文件url的域名blog.huaj.tk也写一条完全一样的rewrite，就可以强行纠正这个傻逼平台的“知错不改”了。

置顶

用谷歌搜索，大多数会推荐hexo-generator-index-pin-top。但是这个插件是远古版本的了，而且实际上现在的官方generator是支持置顶的，只是文档里没写。

看官方generator的generator.js可以看见有这么一行：

因此只要在front matter里面添加sticky: 数值，就会在生成的时候将这个文章置顶。如果有多篇文章设置了sticky，会按sticky数值由大到小排序。

假设B设置了sticky 2，A设置了sticky 1，那么最终就会B出现在A的上面

与Typora搭配使用时的图片链接

经测试可用ZaiZheTingDun的hexo-simple-image，并设置_config.yml的post_asset_folder为true。Typora的全局图片位置路径设置为./${filename}/并勾选优先使用相对路径。

字数统计与预计阅读时间

NexT主题的推荐组件hexo-symbols-count-time并不能使用，出来的效果是字数统计无、预计时间NaN。

好在它的README.md开头的一句提供了改进的灵感：

Better than hexo-reading-time and faster than hexo-wordcount. No external dependencies.

要做的事：安装hexo-wordcount；修改主题代码以显示来自hexo-wordcount的数据。

修改代码首先要把原来由hexo-symbols-count-time提供的数据标签换掉，文件是next主题的 layout/_macro/post.swig，两处修改分别是：

  1. <span>{{ symbolsCount(post) }}</span>改为<span>{{ wordcount(post.content) }}</span>，这是修改文章字数统计的。
  2. <span>{{ symbolsTime(post, config.symbols_count_time.awl, config.symbols_count_time.wpm, __('symbols_count_time.time_minutes')) }}</span>改为</span>{{ min2read(post.content) }}分钟</span>，这是修改阅读时间的。

随后发现，如果保留原来的判断代码，不安装hexo-symbols-count-time是不会显示字数统计和阅读时间的。这里采用了最暴力的方法，删掉 layout/_macro/post.swig的以下代码（基于commit d24c48e，括号内为行号）：

  1. {%- if config.symbols_count_time.symbols %}（132）及其对应的 {%- endif %}（143）
  2. {%- if theme.symbols_count_time.separated_meta %}<br>{%- endif %}（133），只保留<br>
  3. {%- if theme.symbols_count_time.item_text_post %}(138)及其对应的 {%- endif %}（140）
  4. {%- if config.symbols_count_time.time %}（145）及其对应的 {%- endif %}（155）
  5. {%- if theme.symbols_count_time.item_text_post %}（150）及其对应的 {%- endif %}（152）

就可以正常运作了。可以参考我的fork的a1246b5。
